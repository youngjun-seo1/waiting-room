Waiting Room

트래픽 폭주로부터 서비스를 보호하는 대기실 시스템

기술연구소 리더 미팅

이런 상황, 겪어보셨나요?

  • 쿠폰 선착순 이벤트 오픈 → 순간 트래픽 폭주
  • 서버 다운 → 사용자 이탈 → CS 폭주
  • "새로고침 전쟁" → 누가 먼저인지 알 수 없음
놀이공원 줄서기처럼,
서버에 들어갈 수 있는 인원을 제한하고
나머지는 순서대로 대기시키는 시스템

어떻게 동작하나요?

사용자 접속 ↓ [대기실] 자리 있어? ──Yes──→ [서비스 페이지] ↓ No [대기 페이지] "현재 32번째, 약 2분 대기" ↓ (자동 입장) [서비스 페이지]
선착순 FIFO 대기열 실시간 순번 표시 자동 입장 (새로고침 불필요)

어떻게 쓰나요?

1. 코드 변경 없이 적용

[변경 전] 사용자 → 서비스 서버 [변경 후] 사용자 → Waiting Room → 서비스 서버

리버스 프록시 방식 — 서비스 서버 앞에 놓기만 하면 됨

2. 스케줄 기반 자동 운영

  • 이벤트 시작/종료 시간 설정
  • 자동으로 대기실 ON/OFF
  • 수동 개입 없이 운영 가능

3. Admin 대시보드

  • 실시간 모니터링
  • Quick Test 즉시 테스트
